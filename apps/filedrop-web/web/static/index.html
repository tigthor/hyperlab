what actually happens

1 PAKE both sides prove the passphrase with CPace — never sending it
2 CONFIRM wrong guess? the connection dies here. zero file bytes moved
3 MANIFEST name, size, and a BLAKE2b leaf per chunk, sealed with the session key
4 CHUNK 64 KiB at a time — each verified against its leaf ✓ before it is written
5 RECEIPT the receiver signs for the exact bytes and hash — proof-of-delivery, not vibes

why you can trust it

wrong passphrase, nothing moves

The passphrase seeds a CPace exchange. A guesser — or a man-in-the-middle — fails key confirmation before the manifest exists.

every chunk proves itself

The manifest commits to each chunk's hash. A flipped bit anywhere is caught on arrival, and the whole file re-verifies before it is saved.

the relay sees ciphertext

In the browser, a gateway relays DHT packets — but every frame is sealed with a key derived from the passphrase, which the gateway never learns. Its compromise is a denial of service, not a leak.
